# 🕵️‍♂️ TraceDetectiveAndroid

> **"Data never lies. The culprit is in the traces."**
> 안드로이드 성능 지연의 근본 원인을 스스로 추론하고 검거하는 **자율형 AI 포렌식 분석기**

`TraceDetectiveAndroid`는 Perfetto 트레이스 데이터를 기반으로 **Normal 대비 Slow 상황의 차이(Delta)**를 정밀 분석하여 성능 병목의 원인을 찾아내는 AI 에이전트 시스템입니다. 단순한 분석 툴을 넘어, 스스로 가설을 세우고 도구를 운용하는 **자율적 수사 아키텍처**를 지향합니다.

---

## 🌟 Key Features

### 1. 🧠 자율적 수사 루프 (Agentic Workflow)
단순한 텍스트 요약을 넘어, AI가 스스로 가설을 세우고 도구를 선택하여 최대 5라운드 동안 심층 수사를 진행합니다.
* **Thought-Action-Observation:** AI의 추론 과정(Thought)을 실시간으로 노출하여 수사의 투명성 확보.
* **Multi-Tooling:** 상황에 따라 여러 수사 도구를 동시에 호출하여 분석 속도 극대화.

### 2. 🧬 정밀 Delta($\Delta$) 수사 기법
"왜 느린가?"에 대한 답을 찾기 위해 $\Delta T = T_{slow} - T_{normal}$ 데이터를 대조 분석합니다.
* **Causal Chain 추적:** Binder 지연, Lock 경합, GC 발생, CPU 스케줄링 등 복잡한 인과관계를 논리적으로 규명.

### 3. 🛡️ Strategy A: 정직한 수사 프로토콜
환각(Hallucination)을 최소화하고 데이터에 기반한 진실만을 보고합니다.
* **Cold Case 선언:** 데이터가 동일하거나 증거가 부족할 경우 억지로 소설을 쓰지 않고 "분석 불능" 사유를 논리적으로 보고하고 분석를 종결합니다.

## 🏗️ Architecture

* **Brain:** `Gemma 3 12B (Q6_K / Q4_K_M)` - 고성능 로직 추론 및 SQL 설계 엔진.
* **Heart:** 중앙 관제 센터. 분석 라운드 관리 및 예외 처리 제어.
* **Senses:** Perfetto Trace Processor 기반의 실시간 데이터 쿼리 엔진.
* **Manual:** 철저한 지침서.

## 🛠️ Magnificent Seven (Forensic Tools)

성능 지연의 실체를 밝히기 위해 자유자재로 운용하는 7가지 정예 도구입니다.

1. **초동 수사 및 시스템 전반 스캔**
   * 전체 성능의 기준점(Milestone)을 비교하여 지연이 시작된 지점을 특정하고, 시스템 내 가장 '바쁜' 프로세스들을 식별합니다.
2. **메인 스레드 정밀 프로파일링**
   * 대상 앱의 UI 스레드 내 함수 호출 이력을 분석하여, 실제 실행 시간이 비정상적으로 길어진 병목 구간을 색출합니다.
3. **IPC 및 Binder 호출 추적**
   * 프로세스 간 통신(IPC) 과정에서의 대기 시간을 분석하고, 응답을 지연시킨 상대방 프로세스를 식별합니다.
4. **스레드 상태 및 스케줄링 분석**
   * 스레드가 실행(Running) 중이었는지, 자원을 기다리는(Runnable) 중이었는지 분석하여 CPU 경합 원인을 파악합니다.
5. **Lock 경합 및 동기화 분석**
   * 자원을 사용하기 위해 대기하는 Lock 경합 상황을 분석하고, 자원을 점유 중인 '범인(Owner) 스레드'를 추적합니다.
6. **메모리 관리 및 GC 분석**
   * 가비지 컬렉션(GC) 발생 시점과 이로 인해 UI 스레드가 일시 정지(Stop-the-world)된 정확한 시간을 확인합니다.
7. **자율형 커스텀 SQL 수사**
   * AI 직관에 따라 데이터베이스에 직접 질의하여, 표준 도구로 잡히지 않는 특수하고 복잡한 인과관계를 규명합니다.
