#  TraceDetectiveAndroid

> **"Data doesn't lie. The culprit is within the trace."**

`TraceDetectiveAndroid`는 안드로이드 시스템 트레이스(perfetto)를 분석하여
성능 저하(Regression)의 근본 원인을 추적하는 **AI 기반 성능 포렌식 엔진**입니다.
단순한 "느림"을 넘어 **"왜, 어디서, 누구 때문에" 느려졌는지**에 대한 인과관계를 입증합니다.

---

##  Purpose

안드로이드 성능 분석은 수만 개의 슬라이스와 복잡한 스케줄링 데이터 사이에서
'범인'을 찾는 고통스러운 과정입니다. 본 프로젝트는 다음을 목표로 합니다:

1. **Delta (Δ) 기반 분석**
   - 정상(Normal)과 지연(Slow) 트레이스를 대조하여 퇴보 지점을 정밀 타격
2. **인과관계 규명**
   - 메인 스레드 지연이 Binder 통신 때문인지, Lock 경합 때문인지,
     혹은 외부 프로세스의 CPU 가로채기(Steal) 때문인지 밝힘
3. **자율 수사**
   - AI가 가설을 세우고 적절한 도구를 이용하여 스스로 최종 보고서를 작성

---

##  Token Dashboard

자원 소모량을 실시간으로 모니터링합니다. 효율적인 SQL 쿼리 설계로
최소한의 토큰으로 최대의 증거를 확보합니다.

```
Tokens: 12,450 / 32,768 (80%)
``` 
- **Usage**: 모델 컨텍스트 윈도우 점유율

##  Role & Persona

이 엔진은 단순한 스크립트가 아닙니다.

> "데이터는 거짓말을 하지 않죠. 범인은 바로 이 코드입니다."

냉철한 분석과 위트 있는 통찰력을 가진 AI가 당신의 코드 성능을 지킵니다.
